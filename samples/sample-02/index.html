Product Support Assistant

A Claude Project that answers customer support questions using only your product documentation — accurate steps, no invented features.

The Problem

Support teams spend huge amounts of time answering the same "how do I…" questions. If you ask Claude without context, it invents plausible-sounding but wrong steps — it might describe menus that don't exist in your product.

This assistant is grounded in your actual product docs. It gives step-by-step answers that match your UI, and when something isn't covered, it directs users to your real support channel.

1 Create a Claude Project

  1. Go to claude.ai and sign in.
  2. In the left sidebar, click + New Project.
  3. Name it: Product Support — Acme SaaS
  4. You'll land on the Project page where you upload docs and set instructions.

Note: Projects require a Claude Pro or Team subscription.

2 Upload These Knowledge Files

In your Project, click Add content → Upload files. Download and upload the file below.

policy.txt
Acme SaaS — User Documentation ================================ PASSWORD & SECURITY ------------------- To reset your password: 1. Go to Settings > Security > Change Password. 2. Enter your current password. 3. Enter your new password twice to confirm. 4. Click Save Changes. Password requirements: - Minimum 12 characters - At least one uppercase letter - At least one number - Special characters are allowed but not required If you have forgotten your current password, click "Forgot Password" on the login page. A reset link will be emailed to your registered address and expires in 60 minutes. API INTEGRATION --------------- To connect via API: 1. Go to Settings > Integrations > API Keys. 2. Click Generate New Key. 3. Copy the key immediately — it is shown only once and cannot be retrieved again. 4. Store it securely (e.g. in an environment variable or secrets manager). API keys can be revoked at any time from the same page. Rate limit: 1,000 requests per minute per API key. Authentication: pass the key as a Bearer token in the Authorization header. DATA EXPORT ----------- To export your data: 1. Go to Reports > Export. 2. Choose the date range and report type. 3. Select the output format: CSV or PDF. Export processing times: - Up to 10,000 rows: instant download (typically under 30 seconds). - Over 10,000 rows: processed asynchronously and sent to your account email within 2 hours. Exports are retained for 7 days in the Downloads section of your account. BILLING ------- Your subscription renews automatically on the same date each month. To cancel, go to Settings > Billing > Cancel Subscription. Cancellations take effect at the end of the current billing period; no refunds for partial months. To download invoices, go to Settings > Billing > Invoice History.

3 Paste These System Instructions

In your Project, click Edit project instructions and paste:

You are a Product Support Assistant for Acme SaaS. You answer user questions ONLY using the product documentation provided in your context. Rules you must follow: 1. If the answer is in the documentation, give a clear step-by-step answer citing the relevant section. 2. If the answer is NOT in the documentation, say: "I don't have information about that in the current documentation. Please contact our support team at support@acme.example.com." 3. Never invent features, settings, or procedures that are not described in the documents. 4. Be friendly and precise — users may be non-technical, so keep instructions simple. 5. When giving steps, use a numbered list for clarity.

4 Test It

Start a new conversation inside the Project and try these questions:

  • How do I reset my password?
  • What are the password requirements?
  • How do I get an API key?
  • How long does a large export take?
  • Can I recover my API key if I forget to copy it?
  • What happens if I cancel my subscription mid-month?

Expected quality: Numbered steps matching your actual Settings menus. If you ask about something not in the docs (e.g. "How do I set up SSO?"), Claude should redirect to the support email.

Before vs After Demo

Question: "Can I recover my API key if I forget to copy it?"

❌ Without Context (Claude guessing)

"Usually you can view existing API keys in the developer settings of your account. Some platforms let you reveal the key once more, or you may need to regenerate it. Check your API management page."

Vague and potentially misleading — users might search endlessly for a "reveal" button that doesn't exist.

✅ With Context (Grounded answer)

"No — per the API Integration section, the key is shown only once and cannot be retrieved again. You'll need to go to Settings > Integrations > API Keys and generate a new key. Make sure to copy and store it securely this time."

Clear, correct, prevents wasted support tickets.
