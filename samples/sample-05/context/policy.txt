CHAPTER 1: INTRODUCTION TO MACHINE LEARNING
--------------------------------------------
Machine learning (ML) is a branch of artificial intelligence where systems
learn from data to make predictions or decisions without being explicitly
programmed for each task.

The three main paradigms of machine learning:

1. Supervised Learning
   - The model is trained on labelled data (each input has a known correct output).
   - The goal is to learn a mapping from inputs to outputs.
   - Examples: email spam detection, house price prediction, image classification.
   - Common algorithms: linear regression, decision trees, support vector machines.

2. Unsupervised Learning
   - The model finds patterns in data without labels.
   - The goal is to discover structure, groupings, or representations in the data.
   - Examples: customer segmentation, topic modelling, anomaly detection.
   - Common algorithms: k-means clustering, principal component analysis (PCA).

3. Reinforcement Learning
   - The model (called an agent) learns by interacting with an environment.
   - The agent receives rewards for good actions and penalties for bad ones.
   - The goal is to learn a policy that maximises cumulative reward over time.
   - Examples: game-playing AI (chess, Go), robot navigation, recommendation systems.

CHAPTER 2: NEURAL NETWORKS
---------------------------
A neural network is a model inspired by the structure of the human brain.

Architecture:
- Input layer: receives the raw features (e.g. pixel values, numbers).
- Hidden layers: intermediate layers that learn abstract representations.
- Output layer: produces the final prediction (e.g. a class label or a number).

Activation functions:
- Introduce non-linearity into the network, allowing it to learn complex patterns.
- ReLU (Rectified Linear Unit): outputs max(0, x). Fast and commonly used in hidden layers.
- Sigmoid: outputs a value between 0 and 1. Used in binary classification output layers.
- Softmax: outputs a probability distribution over multiple classes.

Training a neural network:
1. Forward pass: input data flows through the network to produce a prediction.
2. Loss calculation: the difference between the prediction and the true label is measured
   using a loss function (e.g. cross-entropy loss for classification).
3. Backpropagation: the gradient of the loss with respect to each weight is computed
   by applying the chain rule backwards through the network.
4. Weight update: weights are adjusted in the direction that reduces the loss
   (gradient descent with a learning rate).

CHAPTER 3: MODEL EVALUATION
----------------------------
Key metrics for evaluating classification models:

Accuracy = (number of correct predictions) / (total number of predictions)
         = (TP + TN) / (TP + TN + FP + FN)

Precision = TP / (TP + FP)
Measures: of all the samples the model predicted as positive, how many were actually positive?
High precision means few false alarms.

Recall (Sensitivity) = TP / (TP + FN)
Measures: of all the samples that were actually positive, how many did the model find?
High recall means few missed positives.

F1 Score = 2 * (Precision * Recall) / (Precision + Recall)
A balanced metric that combines precision and recall.

TP = True Positive | TN = True Negative | FP = False Positive | FN = False Negative

Overfitting vs Underfitting:
- Overfitting: the model memorises the training data and performs well on training data
  but poorly on new, unseen data. Signs: low training loss, high validation loss.
  Fixes: more training data, dropout, regularisation (L1/L2), simpler model.
- Underfitting: the model is too simple to capture the underlying patterns.
  Signs: high training loss and high validation loss. Fix: more complex model, more training.

Train/Validation/Test split:
- Training set: used to train the model (typically 70–80% of data).
- Validation set: used to tune hyperparameters and detect overfitting (10–15%).
- Test set: used once at the end to measure final performance (10–15%).
