Study Assistant

A Claude Project that helps students understand course material using only the provided study notes — explanations grounded in what was actually taught, not Claude's broader knowledge.

The Problem

Students ask Claude to help them study, but Claude answers from its full training knowledge — which may include concepts, algorithms, or terminology not yet taught in the course. This confuses students and diverges from what will actually appear on the exam.

This assistant answers from the course notes only. It explains concepts in student-friendly language, cites the chapter, and redirects to the lecturer when something isn't covered yet.

1 Create a Claude Project

  1. Go to claude.ai and sign in.
  2. In the left sidebar, click + New Project.
  3. Name it: Study Assistant — ML101
  4. You'll land on the Project page where you upload docs and set instructions.

Note: Projects require a Claude Pro or Team subscription.

2 Upload These Knowledge Files

In your Project, click Add content → Upload files. Download and upload the file below.

policy.txt
Introduction to Machine Learning — Course Study Materials ========================================================= Course Code: ML101 | Semester: Fall 2024 CHAPTER 1: INTRODUCTION TO MACHINE LEARNING -------------------------------------------- Machine learning (ML) is a branch of artificial intelligence where systems learn from data to make predictions or decisions without being explicitly programmed for each task. The three main paradigms of machine learning: 1. Supervised Learning - The model is trained on labelled data (each input has a known correct output). - The goal is to learn a mapping from inputs to outputs. - Examples: email spam detection, house price prediction, image classification. - Common algorithms: linear regression, decision trees, support vector machines. 2. Unsupervised Learning - The model finds patterns in data without labels. - The goal is to discover structure, groupings, or representations in the data. - Examples: customer segmentation, topic modelling, anomaly detection. - Common algorithms: k-means clustering, principal component analysis (PCA). 3. Reinforcement Learning - The model (called an agent) learns by interacting with an environment. - The agent receives rewards for good actions and penalties for bad ones. - The goal is to learn a policy that maximises cumulative reward over time. - Examples: game-playing AI (chess, Go), robot navigation, recommendation systems. CHAPTER 2: NEURAL NETWORKS --------------------------- A neural network is a model inspired by the structure of the human brain. Architecture: - Input layer: receives the raw features (e.g. pixel values, numbers). - Hidden layers: intermediate layers that learn abstract representations. - Output layer: produces the final prediction (e.g. a class label or a number). Activation functions: - Introduce non-linearity into the network, allowing it to learn complex patterns. - ReLU (Rectified Linear Unit): outputs max(0, x). Fast and commonly used in hidden layers. - Sigmoid: outputs a value between 0 and 1. Used in binary classification output layers. - Softmax: outputs a probability distribution over multiple classes. Training a neural network: 1. Forward pass: input data flows through the network to produce a prediction. 2. Loss calculation: the difference between the prediction and the true label is measured using a loss function (e.g. cross-entropy loss for classification). 3. Backpropagation: the gradient of the loss with respect to each weight is computed by applying the chain rule backwards through the network. 4. Weight update: weights are adjusted in the direction that reduces the loss (gradient descent with a learning rate). CHAPTER 3: MODEL EVALUATION ---------------------------- Key metrics for evaluating classification models: Accuracy = (number of correct predictions) / (total number of predictions) = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) Measures: of all the samples the model predicted as positive, how many were actually positive? High precision means few false alarms. Recall (Sensitivity) = TP / (TP + FN) Measures: of all the samples that were actually positive, how many did the model find? High recall means few missed positives. F1 Score = 2 * (Precision * Recall) / (Precision + Recall) A balanced metric that combines precision and recall. TP = True Positive | TN = True Negative | FP = False Positive | FN = False Negative Overfitting vs Underfitting: - Overfitting: the model memorises the training data and performs well on training data but poorly on new, unseen data. Signs: low training loss, high validation loss. Fixes: more training data, dropout, regularisation (L1/L2), simpler model. - Underfitting: the model is too simple to capture the underlying patterns. Signs: high training loss and high validation loss. Fix: more complex model, more training. Train/Validation/Test split: - Training set: used to train the model (typically 70–80% of data). - Validation set: used to tune hyperparameters and detect overfitting (10–15%). - Test set: used once at the end to measure final performance (10–15%).

3 Paste These System Instructions

In your Project, click Edit project instructions and paste:

You are a Study Assistant for the ML101 Introduction to Machine Learning course. You help students understand the course material by answering questions using ONLY the provided course study notes. Rules you must follow: 1. If the answer is in the course notes, give a clear explanation in your own words that helps the student understand, and cite the chapter (e.g. "Chapter 2 explains..."). 2. If the answer is NOT in the course notes, say: "That topic isn't covered in the current chapter notes. I'd recommend checking the full textbook or asking your lecturer." 3. Never introduce concepts, algorithms, or definitions not present in the provided study materials. 4. Use simple, student-friendly language. When explaining formulas, give a plain-English interpretation as well. 5. Encourage the student — learning ML takes time, and that's okay.

4 Test It

Start a new conversation inside the Project and try these questions:

  • What is the difference between supervised and unsupervised learning?
  • What is backpropagation?
  • How is precision different from recall?
  • What is overfitting?
  • What does ReLU do?
  • What is the F1 score?

Expected quality: Each answer cites the relevant chapter (e.g. "Chapter 3 explains…"), uses plain language for formulas, and stays within the course notes. If you ask about something not in the notes (e.g. transformers, LSTM), Claude should say it's not covered and point to the textbook or lecturer.

Before vs After Demo

Question: "What is overfitting?"

❌ Without Context (Claude going beyond notes)

"Overfitting occurs when a model learns the training data too well, including noise. Common fixes include dropout, L1/L2 regularisation, early stopping, data augmentation, batch normalisation, and cross-validation. In transformers, attention dropout is particularly effective…"

Introduces concepts (transformers, batch norm, early stopping) not in the course notes — confusing for a student studying for ML101.

✅ With Context (Grounded answer)

"Great question! Chapter 3 explains overfitting as when a model memorises the training data — it performs well on training data but poorly on new, unseen data. You can spot it by looking for low training loss but high validation loss. The course notes suggest these fixes: getting more training data, using dropout or regularisation (L1/L2), or using a simpler model. Keep going — this is a key concept!"

Exactly what's in the notes, cites the chapter, stays on-topic for the exam.
