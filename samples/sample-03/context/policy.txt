Engineering Team — Wiki & Runbooks
====================================

DEPLOYMENT
----------
Staging deployment:
1. Run: npm run build
2. Run: git push origin staging
3. Staging deploys take approximately 3 minutes to complete.
4. Monitor the deployment in #deploys on Slack.

Production deployment:
1. Create a PR from your feature branch to main.
2. Require at least one approval from a senior engineer.
3. After merge, production deploys automatically via CI/CD.
4. Production deploys take approximately 8 minutes.
5. Rollback: run /rollback prod in the #deploys Slack channel.

DATABASE ACCESS
---------------
Connect using TablePlus (preferred) or psql.
Host: db.internal
Port: 5432
Credentials: stored in the team 1Password vault under "Dev DB".
Ask your team lead for access to the 1Password vault on your first day.
Production DB access requires a separate "Prod DB" entry — request from the CTO.

ON-CALL ROTATION
----------------
On-call runs Monday–Sunday, one person per week.
The on-call schedule is posted in the #on-call Slack channel every Friday.
To join the rotation, message @oncall-admin in Slack with your name and preferred weeks.
During on-call, respond to PagerDuty alerts within 15 minutes.
Escalation: if unresolved in 30 minutes, page the CTO.

CODE REVIEW
-----------
All PRs require at least one approval from a senior engineer before merging.
PRs should be under 400 lines of changed code where possible — break up larger changes.
Add a description of what the PR does and why in the PR body.
Use the PR template in .github/PULL_REQUEST_TEMPLATE.md.
Link the relevant Linear ticket in the PR body.
